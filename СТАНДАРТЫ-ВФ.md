# Эталоны «стандарт» — вертикальная ферма

Краткая инструкция для операторов калькулятора Daogreen.

## Откуда берутся цифры

| Режим | Файл-источник | Как считается эталон |
|-------|---------------|----------------------|
| **Каналы** (ВФ, вкладка «каналы») | `АУДИТ/КАНАЛЫ.xlsx` | Прорастание, дни, плотность, интервал — **середина** диапазона из таблицы. **Урожай за срез (г)** — **верх** диапазона **+12,5%**. |
| **Поддоны 130×65** (ВФ, вкладка «поддоны») | `АУДИТ/ПОДДОНЫ.xlsx` | Числа из диапазона — **между серединой и верхней границей** (как в таблице аудита). |
| **Цветы** | `АУДИТ/ЦВЕТЫ.xlsx` | Отдельная таблица; урожай часто в **штуках**, частичная срезка. |

**Теплица** (кнопка «Теплица») — другие сорта и модель, эти файлы **не используются**.

## Кнопка «стандарт»

Рядом с полем (прорастание, дни, масса, плотность, интервал срезки…):

- **стандарт** — подставить значение из таблицы агронома для выбранного сорта;
- после изменения ползунка бейдж показывает, что значение **ваше**, не эталон.

Сетка **«Стандарты VF»** (чекбоксы): какие поля всегда держать на эталоне, какие можно менять вручную.

## Можно ли менять параметры?

**Да.** Эталон — стартовая точка. Ползунки, ручная масса, своя плотность — для вашего сценария. В экономику можно импортировать либо эталон с посадки, либо то, что получилось после правок.

## Примеры (каналы)

Если в таблице урожай **15–20 г**:

- середина = 17,5 г (для подсказок по другим полям не используется для урожая каналов);
- **эталон в калькуляторе** = верх 20 г × 1,125 ≈ **23 г** за срез.

## Обновление эталонов

При смене таблиц в папке `АУДИТ/` разработчик запускает `npm run sync:audit` (скрипт `_tools/sync-vf-audit-xlsx.js`), затем `npm run build`. Отчёт «было → стало» — в `АУДИТ/sync-report.md`. Номер сборки — внизу экрана (зелёная плашка).

В калькуляторе на вкладке посадки ВФ: блок **«Откуда цифры»** (сроки цикла) и подсказка у поля **«Вес за срез»** при ручном вводе.

## Вопросы

Если цифра «стандарта» не совпадает с вашей таблицей — пришлите название сорта, вкладку (каналы/поддоны) и скрин строки из Excel.
