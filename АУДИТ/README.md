# Папка АУДИТ — эталонные замеры по видам зелени

Здесь хранятся **проверенные на практике параметры** выращивания: сроки, плотность, урожай за срез, интервалы срезки. Это не черновики, а рабочие стандарты Daogreen, из которых собираются справочники калькулятора.

## Что в репозитории

| Файл | Назначение |
|------|------------|
| `sync-report.md` | Отчёт последней синхронизации xlsx → JS |
| `vf-cultivars.js` (в корне) | Сгенерированный справочник каналов ВФ |
| `pallet-cultivars.js` (в корне) | Сгенерированный справочник поддонов |

## Исходные таблицы (локально, не в git)

Для пересборки справочников положите в эту папку:

| Файл | Результат |
|------|-----------|
| `КАНАЛЫ.xlsx` | `vf-cultivars.js` |
| `ПОДДОНЫ.xlsx` | `pallet-cultivars.js` |
| `ЦВЕТЫ.xlsx` | секция цветов в обоих справочниках |

Команда:

```bash
npm run sync:audit
```

## Правила разбора диапазонов

| Среда | Урожай | Остальные поля |
|-------|--------|----------------|
| Каналы ВФ | **верх диапазона + 12,5%** (г; для `шт` — без ×1,125) | середина |
| Поддоны | **midUpper** | **midUpper** |
| Цветы | по `ЦВЕТЫ.xlsx`, часто **шт** | по таблице |

На хостинг **xlsx не нужны** — достаточно сгенерированных `vf-cultivars.js` и `pallet-cultivars.js`. После `sync:audit` выполните `npm run build` перед выкладкой.
