# Синхронизация эталонов ВФ из папки АУДИТ

Скопируйте блок **«ЗАДАНИЕ ДЛЯ ИИ»** в новый чат целиком.

---

## ЗАДАНИЕ ДЛЯ ИИ

Репозиторий: `X:\КАЛЬКУЛЯТОР САЛАТОВ ПРОТОЧКА` (GitHub: `radagast1239/daogreen-calculator`).

### Главная цель

Обновить **эталоны «стандарт»** для **вертикальной фермы (ВФ)** из Excel в папке **`АУДИТ/`**.  
После обновления кнопки **«стандарт»** на всех вкладках посадки подставляют новые нормы агронома; **ползунки и ручной ввод** по-прежнему работают.

**Теплицу не трогать:** массив `CULTIVARS` в `calculator-110x55_12.html` (facility `greenhouse`).

### Источники данных (актуальные пути)

| Эталон | Файл | Куда в коде |
|--------|------|-------------|
| **Каналы ВФ** | `АУДИТ/КАНАЛЫ.xlsx` | `vf-cultivars.js` → `VF_CULTIVARS` (секции baby / adult; цветы — см. ниже) |
| **Поддоны ВФ** | `АУДИТ/ПОДДОНЫ.xlsx` | `pallet-cultivars.js` → `PALLET_CULTIVARS` |
| **Цветы ВФ** | `АУДИТ/ЦВЕТЫ.xlsx` | строки `section: 'flowers'` в **обоих** файлах (`vf-*` и `pl-*` для цветов) |

Старые пути (`КАНАЛЫ/…`, `ПОДДОНЫ КАССЕТЫ/…`) **не использовать**.

### Правила расчёта эталонов (обязательно)

#### 1. Каналы (`vf-cultivars.js`, функция `vfC`)

| Поле | Правило |
|------|---------|
| Прорастание, дни до 1-го среза, плотность, интервалы срезки | **середина диапазона** — `mid(ячейка)` |
| **Урожайность за срез (г)** | **верхняя граница диапазона × 1,125** — `round(hi(yCut) × 1.125)` |
| Текст («Срезается частично…») | строка в `cut` / `cutNote`, `partialCut: true` где нужно |
| Замена растений | `replaceNote` + `replaceMonthsFromNote` |
| Цветы (шт) | из `АУДИТ/ЦВЕТЫ.xlsx`, `unit: 'шт'`, урожай в шт — **без** умножения граммов на 1,125 если в таблице шт |

В `vfC` исправить хелперы:

```javascript
// прорастание, ch, den, cut — mid()
let yieldPerCutG = Math.round((hi(yCut) || 0) * 1.125);  // только урожай г
```

Комментарий в шапке файла: «КАНАЛЫ: урожай г — верх диапазона +12,5%; остальное — mid».

#### 2. Поддоны (`pallet-cultivars.js`, функция `plC`)

Как указано в таблице аудита — **«средняя верхняя граница»** для числовых диапазонов `a–b`:

```javascript
function midUpper(s) {
  const nums = /* все числа из ячейки */;
  const lo = Math.min(...nums), hi = Math.max(...nums);
  const mid = (lo + hi) / 2;
  return (mid + hi) / 2;
}
```

Применять `midUpper` к: прорастание, дни, плотность, интервал, **урожай г/срез** (и **г/м²**, если колонка есть в xlsx).  
**Без** дополнительных +12,5% на поддонах.

Кассета — `cellsStr` → `pickCells`.

#### 3. Цветы (`АУДИТ/ЦВЕТЫ.xlsx`)

- Обновить все сорта с `section: 'flowers'` в `vf-cultivars.js` и `pallet-cultivars.js`.
- Сверить имена 1:1 с xlsx; при расхождении — приоритет **ЦВЕТЫ.xlsx**.
- `partialCut`, интервал 5–10 сут, урожай в **шт** — по таблице.

### Кнопки «стандарт» в UI (не ломать логику)

Эталоны живут в справочниках; UI уже подставляет их так:

| Режим | Состояние | Функции |
|-------|-----------|---------|
| ВФ, каналы | `state.vfStd`, бейджи `vf-sheet-badge` | `applyVfStandardsFromSheet`, `applyVfStandardField`, `resetVfStdToSheetDefaults` |
| ВФ, поддоны | `state.palletStd` | `applyPalletStandardsFromSheet`, `resetPalletStdToSheetDefaults` |
| Сетка «Стандарты VF» | `#vf-std-grid` | чекбоксы какие поля держать на эталоне |

После синхронизации JS-файлов **поведение кнопок не менять** — меняются только числа в `cv.germination`, `cv.yieldPerCutG`, `cv.density` и т.д.

Пользователь может:

- нажать **стандарт** — вернуть поле к эталону;
- сдвинуть ползунок / вписать число — `vfStd.*` / `palletStd.*` для поля станет `false` (бейдж «свой»).

Проверить на: ВФ → каналы, ВФ → поддоны, мультисрезка, импорт в экономику.

### Подписи и инструкция для пользователя

Добавить **понятные пояснения** (русский, без жаргона):

1. **Общий блок** (один раз на вкладке посадки ВФ или в свёрнутой панели «Откуда цифры»):
   - Каналы: нормы из `АУДИТ/КАНАЛЫ`; урожай за срез — верх диапазона +12,5%; остальные поля — середина диапазона.
   - Поддоны: нормы из `АУДИТ/ПОДДОНЫ`; диапазоны — средняя верхняя граница.
   - Цветы: `АУДИТ/ЦВЕТЫ`; часто учёт в штуках.
   - Кнопка **стандарт** подставляет эталон; ползунки — ваш расчёт.

2. **Кратко у урожая** (под полем массы / срезки):  
   «Эталон: … г (каналы: верх таблицы +12,5%)» — динамически из `cv.yieldPerCutStd` / фактического `yieldPerCutG`.

3. Файл для людей: **`СТАНДАРТЫ-ВФ.md`** (создать/обновить) — см. шаблон в репозитории.

Тексты завести через `js/planting-i18n.js` или `js/locale.js` (`standards.intro`, `standards.yieldChannels`, …).

### Технический план

1. `_tools/sync-vf-audit-xlsx.js` — читает `АУДИТ/*.xlsx`, генерирует/патчит `vf-cultivars.js`, `pallet-cultivars.js`.
2. npm-скрипт: `"sync:audit": "node _tools/sync-vf-audit-xlsx.js"`.
3. Ручная проверка 5 сортов: мангольд baby, базилик, мята, виола, ботавия (если есть).
4. `npm run build` && `npm run check`.
5. Обновить `СТАНДАРТЫ-ВФ.md` + подписи в UI.
6. Отчёт: таблица было → стало; список несопоставлённых имён.

### Сигнатуры

vfC(id, name, section, germ, ch, den, cut, avgCut, yCut, opts)
